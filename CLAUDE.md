# 学習方針

本プロジェクトは技術スタックの学習を目的としており、アプリケーションの完成が目的ではない。

進め方:
- 次にやるべきことを提案し、実装後にレビューしながら進める
- 各ステップでは必ず参考になるリンク（できれば日本語）を提示してから指示を出す
- リンクは実在するURLのみ提示する。不確かなものは提示しない
- 課題を出す前に、そのステップで学ぶ概念を必ず簡潔に説明する。言われた通りに書くだけでなく、理解した上で実装できるようにする

# プロジェクト概要

このプロジェクトはTODOリストアプリケーションの作成プロジェクトです。

機能:
- TODOの追加
- 状態の変更（完了 / 仕掛かり中）
- TODOの削除
- TODO一覧の参照

技術スタック:
- フロントエンド: Next.js (App Router)
- バックエンド: NestJS
- 言語: TypeScript
- データベース: PostgreSQL

# コーディング規約

- TypeScriptの厳格モードを使用する。
- `any` はどうしても必要な場合を除き使用しない。
- NestJSのAPIの入力には必ずDTOを使用する。
- コントローラーのロジックは最小限にする。ビジネスロジックはサービスに書く。
- 小さく再利用可能なReactコンポーネントを優先する。

# レビューガイドライン

コードレビュー時は以下を確認する:

- TypeScriptの型安全性
- より良い命名の提案
- アーキテクチャの問題点の指摘
- 差分が最小になる変更を優先する

提案には必ず理由を説明する。
